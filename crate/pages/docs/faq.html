Q: What platform does Crate run on?

Crate is written in Java and therefore platform independent (tested on Linux systems, OSX and Windows). Crate only needs a working Java 7 virtual machine. We recommend using Oracleâ€™s Java on OSX and OpenJDK on Linux Systems. Check out the Installation section in our documentation.

Q: How do I install Crate?

Crate is easy to install. Visit the Download page for instructions.

Q: What do I need to consider when setting up a Crate cluster?

Crate is a distributed datastore by design so in a production environment you usually have a cluster of 3 or more nodes. We at Crate.IO try to make the cluster setup as easy as possible. You can find a step-by-step guide in our documentation.

Q: Is there a limit on how many nodes a Crate cluster can have.

No. You can add as many instances as you want to the cluster. The more nodes, the more powerful the cluster gets!

Q: Is there a limit of memory I can use?

No. We recommend to set the maximum heap size so that there is still space for the file system cache. However it is not recommended to exceed a maximum heap size of 32GB, because more would disable the Java pointer compression of the JVM. For more information please check the Oracle Documentation and the Configuration section in our documentation. If you want to increase the total memory of the cluster you should to add more nodes.

Q: How can I inspect my data?

You can query your own data, the information schema and other system tables using SQL.

Crate also ships with a graphical user interface – or short Admin UI – to inspect not only your data but also the current state of your cluster. It is accessible from the browser on every node on port 4200 at the URL /admin/ (e.g. http://127.0.0.1:4200/admin/ for a local installation).

There is also a command line interface for Crate called crash written in Python. For installation instructions check out the Github project and read the documentation for usage.

Q: What drivers are there that I can use in my application?

Crate has drivers in several programming languages. Check out the Download page for a complete list of libraries.

Q: What is the best way to migrate my data from MongoDB to Crate?

We provide a best practice how to migration your data from MongoDB to Crate.

Q: What is the best way to migrate my data from MySQL to Crate?

We also have a best practice how to migration from MySQL to Crate.

Q: Does Crate do transactions?

No, Crate does not support transactions but does support Optimistic Concurrency Control which allows conflict resolution on the client side.

Q: Does Crate support JOINs?

Not yet. JOINs are on our roadmap and we try to do them the right way in usual quality and well performing.
For a lot of use cases there are other ways to achieve the same result as with JOINs. The best way to start is to take a look at the ARRAY and OBJECT data types to denormalise your data.

Q: Does Crate have a built-in user/permission management?

No. You will need to secure your Crate installation/cluster with the administration weapons of your choice, e.g. firewall rules. You find information about that in the documentation about Multi Node Setup.
