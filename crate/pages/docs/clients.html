Our Java distributions are hosted on Bintray and available through JCenter. The JDBC driver requires Crate 0.38.0 or newer.

  • Crate Java Client (crate-client) Documentation
  • Crate JDBC Driver (crate-jdbc) Documentation

Our Python projects are hosted on Python Packaging Index (PyPI):

  • Crate Python Client (including SQLAlchemy Integration) Documentation
  • Crate Shell (CraSh) Documentation

Our Ruby distributions are hosted on RubyGems. The initial Ruby driver and Activerecord adapter were written by Christoph Klocker.

  • CrateRuby (RubyGems, Github)
  • Activerecord Crate Adapter (RubyGems, Github)

Our PHP packages are hosted on Packagist. There are a PDO driver and a DBAL Driver that depends on the PDO driver.

  • Crate PDO (Packagist, Github)
  • Crate DBAL (Packagist, Github)

Alexander Jarvis (TravelAI) has written a Scala driver and Play plugin for Crate.

  • Crate Scala (Github)
  • Play Crate (Github)

There are several drivers for Node.js written by community distributors. They are available via npm.

  • node-crate by @megastef (npm, Github)
  • crate-connect by @herenow (npm, Github)
  • cratejs by @herenow (npm, Github)

Craterl is a full-blown Crate Erlang client written by Matthias Wahl. Craterl brings full SQL support including parameters and bulk operations, full BLOB support, for GET, PUT, DELETE and HEAD requests.

  • craterl (Github)

crate-mono is a Mono/.NET client driver implementing the ADO.NET interface for Crate written by Mathias Fussenegger.

  • crate-mono (Github)

herenow wrote a Golang driver for Crate that implements the Go sql interface package.

  • go-crate (Github)

Mathias Fussenegger forked dbext to support Crate.

  • dbext.vim (Github)
